# 📘 คู่มือคำสั่ง Git และเครื่องมือสำหรับโปรเจค GD5-Prototype

คู่มือนี้รวบรวมคำสั่งที่จำเป็นสำหรับการพัฒนาและอัพเดทโค้ดขึ้น GitHub ตามเงื่อนไขของโปรเจคที่ระบุว่า **"อัพเดทข้อมูลขึ้น github หลังแก้ไข code เรียบร้อยทันที"**

---

## 🔥 1. คำสั่งแบบรวบตึง (แนะนำให้ใช้บ่อยที่สุด)

หลังจากคุณแก้ไขหรือเพิ่มไฟล์ในโปรเจคเสร็จแล้ว สามารถก๊อปปี้คำสั่งนี้ไปวางใน Terminal เพื่อบันทึกและอัพเดทขึ้น GitHub ในคลิกเดียวได้เลยครับ:

```bash
git add . && git commit -m "update code" && git push
```

> **💡 คำแนะนำ:** คุณสามารถเปลี่ยนคำว่า `"update code"` เป็นรายละเอียดสั้นๆ ของสิ่งที่คุณเพิ่งทำเสร็จ เช่น `"แก้ไขหน้า UI assignee-action-list"` หรือ `"เพิ่มปุ่มกลับหน้าแรกใน logo"`

---

## 📖 2. คำสั่ง Git แยกทีละขั้นตอน (อธิบายการทำงาน)

หากต้องการค่อยๆ ตรวจสอบงานทีละขั้นตอน สามารถใช้คำสั่งแยกตามลำดับดังนี้:

| คำสั่ง | ความหมาย / หน้าที่ |
| :--- | :--- |
| `git status` | **ตรวจสอบสถานะไฟล์** ว่ามีไฟล์ไหนถูกแก้ไข (`M`), สร้างใหม่ (`U`) หรือถูกลบไปบ้าง |
| `git add .` | **เตรียมบันทึกทุกไฟล์** ที่มีการเปลี่ยนแปลงเข้าสู่คิว (Staging Area) |
| `git add <ชื่อไฟล์>` | **เตรียมบันทึกเฉพาะบางไฟล์** เช่น `git add assignee-action-list.html` |
| `git commit -m "ข้อความ"` | **บันทึกประวัติลงเครื่อง** พร้อมระบุข้อความเตือนความจำว่าเราแก้อะไรไป |
| `git push` | **ส่งข้อมูลขึ้น GitHub** เพื่ออัพเดทโค้ดบนเซิร์ฟเวอร์ให้เป็นเวอร์ชันล่าสุด |

---

## 🛠️ 3. คำสั่ง Git สำหรับการทำงานร่วมกันและดูประวัติ

| คำสั่ง | ความหมาย / หน้าที่ |
| :--- | :--- |
| `git pull` | **ดึงโค้ดล่าสุดจาก GitHub** ลงมาที่เครื่องเรา (ควรทำก่อนเริ่มงานทุกวัน กรณีทำงานหลายเครื่องหรือหลายคน) |
| `git log --oneline -5` | **ดูประวัติการ commit ย้อนหลัง 5 รายการล่าสุด** แบบบรรทัดเดียวสั้นๆ |
| `git diff` | **ดูรายละเอียดจุดที่แก้ไข** ว่าบรรทัดไหนถูกเพิ่มหรือลบออกไปบ้างก่อนทำการ commit |

---

## 🚨 4. คำสั่งแก้ปัญหาฉุกเฉิน (Troubleshooting)

| คำสั่ง | ความหมาย / หน้าที่ |
| :--- | :--- |
| `git restore <ชื่อไฟล์>` | **ยกเลิกการแก้ไขไฟล์นั้น** ให้กลับไปเป็นเหมือนตอนที่ commit ล่าสุด (ระวัง: ข้อมูลที่แก้ล่าสุดจะหายไป) |
| `git restore .` | **ยกเลิกการแก้ไขทุกไฟล์** ที่ยังไม่ได้ commit กลับไปค่าเดิมทั้งหมด |
| `git commit --amend -m "ข้อความใหม่"` | **แก้ไขข้อความ commit ล่าสุด** กรณีที่พิมพ์ข้อความอธิบายผิดและยังไม่ได้ push |

---

## 🌐 5. คำสั่งรันเซิร์ฟเวอร์จำลองเพื่อทดสอบหน้าเว็บ (Local Server)

เนื่องจากโปรเจคนี้เป็นไฟล์ HTML / JS หากต้องการเปิดดูหน้าเว็บในเครื่องแบบจำลองเซิร์ฟเวอร์ (เพื่อป้องกันปัญหาเรื่อง CORS หรือ path ไฟล์) สามารถใช้คำสั่งเหล่านี้ได้ครับ:

```bash
# ถ้าติดตั้ง Python ไว้ สามารถรันเซิร์ฟเวอร์ที่พอร์ต 8000
python -m http.server 8000
```
*(จากนั้นเปิดเบราว์เซอร์แล้วเข้า `http://localhost:8000`)*

หรือถ้าใช้ Node.js / npx:
```bash
npx serve .
```
